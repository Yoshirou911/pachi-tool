# パチスロ統合分析ツール 設計図

設定推測・収支管理・店傾向分析を1つに統合し、Python で本格的なデータ分析をしつつ、スマホ/PC どちらからでも使えるツールの設計。Claude Code で段階的に実装していく前提でまとめている。

---

## 1. 基本コンセプト

精度の核は**ベイズ推定**。3つの機能はバラバラではなく、ベイズの枠組みで一本に繋がる。

```
店傾向分析  ──(事前分布を生成)──▶  設定推測エンジン  ──▶  期待値/撤退判断
     ▲                                      │
     └──────────(推測結果を蓄積)◀────────────┘
                       収支・履歴管理
```

- 設定推測エンジンが各台の事後分布(設定1〜6の確率)を出す
- その結果を履歴に蓄積する
- 蓄積データから「この店のこの条件では高設定が出やすい」という**事前分布**を学習する
- 学習した事前分布を次回の推測に注入すると、同じ観測でも精度が上がる

これが「全部入りにすると精度が上がる」理由。3機能が循環してデータが効いてくる。

---

## 2. 技術スタック(スマホ/PC両対応の実現方法)

「Pythonで本格分析」と「スマホでもPCでも」を両立する王道構成:

| 層 | 技術 | 役割 |
|---|---|---|
| 推定コア | Python(numpy/scipy) | ベイズ推定・統計分析。**完成済み: `core/bayes_engine.py`** |
| API | FastAPI | コアをHTTP経由で叩けるようにする |
| DB | SQLite(将来Postgres) | 機種マスタ・実戦履歴・店データ |
| フロント | レスポンシブWeb(React等) | スマホ/PC共通のUI。ブラウザで動くので両対応が自動で実現 |

ポイント: **ネイティブアプリを2つ作らない**。レスポンシブWebなら1つのコードでスマホ・PC両対応になり、入力もスマホで完結する。重い分析はサーバ側のPythonが担当する。

```
├── value/                    期待値・撤退判断(これから)
│   └── ev.py                 事後分布×機械割で期待値を計算
├── api/                      FastAPI(これから)
│   └── main.py
└── web/                      レスポンシブフロント(これから)
```

## 4. 各モジュールの中身

### records(収支・履歴管理)
1回の実戦につき記録する項目: 日付、店、機種、台番号、総ゲーム数、各要素の観測回数、投資、回収、差枚、推測した事後分布。これが全分析の燃料になる。台番号の末尾・角台かどうか・曜日・イベント日フラグなども一緒に持たせると、後で店傾向が読める。

### hall(店傾向分析)
履歴を条件(店×曜日×イベント日×機種×台位置 等)でグループ化し、各条件での「高設定が出た割合」を集計して**事前分布**を作る。サンプルが少ない条件は一様分布に近づける(ベータ分布などで平滑化)と過学習を防げる。出力は `bayes_engine.estimate(obs, prior=...)` にそのまま渡せる形。

### value(期待値・撤退判断)
事後分布と設定別機械割(`machine_kw`)を掛けて期待値を出す。「今の推測なら回し続けるべきか撤退か」を判定。**ツールの実利益はここが大きい** — 高設定を当てる以上に、低設定での粘りを止めて損失を減らす効果が効く。

### api / web
FastAPI で各モジュールをエンドポイント化し、フロントから叩く。スマホでカウントを入力 → サーバで推定 → 結果を即表示、という流れ。

---

## 5. 精度を最高峰にするための要点

1. **多要素同時推定** — 要素が増えるほど事後が尖る。実装済み。記録できる設定差要素はできるだけ多く取る。
2. **正確な理論値** — `data/machines/` の数値の正確さが精度の上限を決める。ここが甘いと他が完璧でも精度は出ない。
3. **店傾向を事前分布に注入** — ベイズの最大の強み。同じ観測でも事前が効けば精度が上がる。
4. **設定変更・宵越し対応** — 朝イチからのデータか、途中からの拾いデータかを区別して扱う。
5. **平滑化** — サンプルの少ない店条件は事前を一様寄りにして過信を防ぐ。

## 7. 現実的な前提(重要)

このツールは「必ず勝てる」装置ではない。最終的な勝敗を決めるのは推測精度ではなく、**その店が高設定をどれだけ使っているか**。高設定が存在しない日はどれだけ精密でも勝てない。

ツールの本当の価値は2つ:
- **低設定を早く見切って損失を減らす**(撤退判断)
- **データで店の傾向を掴み、勝負所を絞る**(事前分布)

期待値プラスの立ち回りは可能だが、それは「精密な推測 × 高設定を使う店の発見 × 規律ある撤退」の合わせ技として成立する。ツールは判断を助けるが、規律を守るのは人間の役目。資金管理を決め、負けても深追いしないルールを最初に決めておくこと。
